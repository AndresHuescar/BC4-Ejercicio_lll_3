Ejercicio 3 

La organización de la Vuelta Ciclista ha decidido comenzar a registrar pequeñas incidencias 
que puedan ocurrir durante una etapa, como caídas, averías o penalizaciones. Para ello 
necesita ampliar la base de datos y desarrollar una herramienta que permita gestionarlas. 

Se pide realizar las siguientes tareas: 

1. Crear la tabla INCIDENCIA
   
Añade una nueva tabla llamada INCIDENCIA al esquema existente.

La tabla debe cumplir las siguientes especificaciones: 

• id_incidencia → entero, clave primaria generada automáticamente 

• id_ciclista → entero, obligatorio, referencia a ciclista(id_ciclista) 

• numero_etapa → entero, obligatorio, referencia a etapa(numero) 

• tipo → texto corto (VARCHAR2(50)), indicando el tipo de incidencia 

Esquema sugerido: 

CREATE TABLE incidencia ( 

id_incidencia   INTEGER GENERATED BY DEFAULT AS IDENTITY PRIMARY 

KEY, 

id_ciclista    

INTEGER NOT NULL, 

numero_etapa    INTEGER NOT NULL, 

tipo  

VARCHAR2(50), 

CONSTRAINT inc_ciclista_fk 

FOREIGN KEY (id_ciclista) REFERENCES ciclista(id_ciclista), 

CONSTRAINT inc_etapa_fk 

FOREIGN KEY (numero_etapa) REFERENCES etapa(numero) 

); 

2. Implementar un programa Java que gestione las incidencias 

Crea un programa Java que se conecte a la base de datos Oracle mediante JDBC y que 
incluya un menú de opciones para gestionar la información de la tabla INCIDENCIA. 

El programa debe ofrecer al menos las siguientes funcionalidades: 

A) Insertar una nueva incidencia 

El programa debe: 

1. Solicitar al usuario:
    
o id del ciclista 

o número de etapa 

o tipo de incidencia 

2. Verificar que el ciclista existe en la tabla CICLISTA.
 
3. Verificar que la etapa existe en la tabla ETAPA.
  
4. Insertar la incidencia en la tabla utilizando PreparedStatement.
 
5. Realizar la operación dentro de una transacción:
   
o Si algo falla → ROLLBACK 

o Si todo es correcto → COMMIT 

B) Mostrar todas las incidencias 

Debe realizarse una consulta con JOIN para mostrar:

• id de incidencia 

• nombre del ciclista 

• número de etapa 

• tipo de incidencia 

Los resultados deben mostrarse con un formato claro, por ejemplo: 

[1] João Almeida – Etapa 3 – Caída 

[2] Chris Harper – Etapa 1 – Avería 

C) Mostrar las incidencias de un ciclista concreto 

El usuario introduce un id_ciclista y el programa muestra todas sus incidencias 
ordenadas por número de etapa.
